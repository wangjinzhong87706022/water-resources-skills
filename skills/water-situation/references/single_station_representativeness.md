# 单站代表性检查

> **适用范围**：`water-situation` skill 中涉及水位均值统计、跨站对比的场景
> **数据来源**：`sl323.st_stbprp_b`（测站属性表）、业务规则文件
> **核心结论**：本库测站密度偏低，洪泽湖/长江各仅 1 站，"测站权重差异导致均值偏倚"的担忧**不成立**，但"单站代表性"需要评估

## 2. 为什么"测站权重"方案被推翻

### 2.1 原方案假设

**原方案认为**：
- 洪泽湖有"多测点"，长江有"数十个干流+支流测点"
- 不同水域测站数量差异大，直接算术平均会稀释主干站权重
- 需要建立"测站权重"知识库，提供加权平均方法

### 2.2 数据库实证

**实测数据**：
| 水域 | 测站数 | 等权均值 | 站均均值 | 差值 |
|------|--------|---------|---------|------|
| 洪泽湖 | 1 | 13.148m | 13.148m | **0.000** |
| 长江 | 1 | 9.528m | 9.528m | **0.000** |
| 里运河 | 4 | 6.779m | 6.779m | **0.000** |

**结论**：
- ❌ **测站数量差异不大**（最大比值 4:1），不存在"数十个测点"
- ❌ **等权平均与加权平均结果相同**（差值 = 0.000m），加权偏倚**不发生**
- ✅ **所有测站采样频率一致**（年均约 7300 条/站），数据量均衡

### 2.3 为什么原方案被推翻

**原方案基于错误假设**，与数据库实况不符：
1. **错误假设1**：洪泽湖/长江有多测点 → 实际各 1 站
2. **错误假设2**：测站数量差异大 → 实际最大比值 4:1
3. **错误假设3**：加权偏倚存在 → 实际等权=加权

**因此，"测站权重"知识库不能解决本库的实际问题。**

## 3. 真正的偏倚风险：单站代表性

### 3.1 什么是单站代表性

**定义**：单站数据是否足以代表整个水域的水位特征。

**示例**：
- 洪泽湖仅蒋坝 1 站 → 蒋坝站能否代表整个洪泽湖的平均水位？
- 长江仅大通(二) 1 站 → 大通站能否代表长江江苏段的水位？

**核心问题**：
- **不是"加权"问题**（测站少，不存在权重差异）
- **而是"代表性"问题**（这 1 个站是不是关键控制站？）

### 3.2 代表性评估清单

在输出单站均值统计结果时，**必须评估并说明**以下问题：

#### ✅ 检查项

- [ ] **该站是否为控制站/代表站？**
  - 查询 `st_stbprp_b.sttp` 字段（站类型）
  - 查询业务规则 `business_rules.md` 中的重点测站映射
  - 若站名含"代表站"、"控制站"、"总站"等关键词，代表性较高

- [ ] **该站是否在流域关键位置？**
  - 入湖/出湖口
  - 干流与支流交汇处
  - 省界/流域边界

- [ ] **该站数据质量是否良好？**
  - 缺测率是否低于阈值（如 5%）
  - 是否有异常值/传感器故障记录

- [ ] **是否有历史对比数据？**
  - 该站是否有多年同期数据
  - 能否与历史均值对比

#### ❌ 不能仅凭测站数量判断

- **错误逻辑**："只有 1 个站，所以结果不可信"
- **正确逻辑**："蒋坝站是洪泽湖湖水控制站，位于入湖口，数据连续性好，单站代表性较高"

### 3.3 代表性评估示例

#### ❌ 错误示范（不提代表性）

```markdown
洪泽湖 2024 年年均水位 13.148m（仅蒋坝 1 站）
```

**问题**：未说明蒋坝站代表性，读者无法判断结论可靠性。

#### ✅ 正确示范（前置说明）

```markdown
洪泽湖 2024 年年均水位 13.148m（**蒋坝站，位于洪泽湖入湖口，为湖流控制站，单站代表性较高**）

⚠️ **局限性说明**：
- 本库洪泽湖仅蒋坝 1 站报送
- 若需评估全湖平均水位，建议补充湖心、出湖口等补充测站数据
- 当前结论主要反映蒋坝站所在入湖口区域的水位特征
```

## 4. 单站代表性检查流程

### 4.1 查询前

```python
def check_station_count(rvnm: str) -> dict:
    """检查水域测站数量"""

    sql = f"""
    SELECT COUNT(DISTINCT b.stcd) as station_count,
           GROUP_CONCAT(DISTINCT b.stnm) as station_names,
           GROUP_CONCAT(DISTINCT b.sttp) as station_types
    FROM sl323.st_stbprp_b b
    JOIN sl323.st_river_r r ON b.stcd = r.stcd
    WHERE b.rvnm = '{rvnm}'
      AND r.tm >= DATE_SUB(CURDATE(), INTERVAL 1 YEAR)
    """

    result = query(sql)[0]
    count = result['station_count']

    return {
        'station_count': count,
        'station_names': result['station_names'],
        'station_types': result['station_types']
    }
```

### 4.2 查询后

```python
def assess_representativeness(rvnm: str, station_count: int) -> str:
    """评估单站代表性并生成说明文字"""

    if station_count >= 5:
        # 多站情况
        return f"✅ {rvnm} 有 {station_count} 个测站，多站均值代表性较高"

    elif station_count == 1:
        # 单站情况：需进一步评估
        sql = f"""
        SELECT b.stnm, b.sttp, COUNT(*) as data_count
        FROM sl323.st_stbprp_b b
        JOIN sl323.st_river_r r ON b.stcd = r.stcd
        WHERE b.rvnm = '{rvnm}'
        GROUP BY b.stcd, b.stnm, b.sttp
        """
        row = query(sql)[0]
        stnm = row['stnm']
        sttp = row['sttp']
        n = row['data_count']

        # 评估是否为控制站（需结合业务规则）
        is_control = check_if_control_station(stnm)  # 自定义函数

        if is_control:
            return f"⚠️ {rvnm} 仅 {stnm} 1 站（{sttp}），为流域控制站，单站代表性较高，但需注意空间局限性"
        else:
            return f"⚠️ {rvnm} 仅 {stnm} 1 站（{sttp}），单站代表性受限，结论需谨慎使用"

    else:
        # 2-4 站
        return f"⚡ {rvnm} 有 {station_count} 个测站，代表性中等"
```

## 6. 常见误区

### 6.1 误区1："测站越多越好"

**错误理解**：测站越多，均值越准确。

**实际情况**：
- 测站质量（是否为控制站）比数量更重要
- 1 个控制站的代表性可能 > 10 个偏僻小站
- 本库中洪泽湖 1 站（蒋坝，控制站）的代表性可能优于某些有 5 站但均为偏僻站的水域

### 6.2 误区2："测站少 = 结果不可信"

**错误理解**：只有 1 个站，结果一定不准。

**实际情况**：
- 若该站是流域控制站、数据质量好、历史连续性强，单站结论可信
- 单站结论的问题在于**空间局限性**（只能代表该站附近区域），而非"统计不准确"

### 6.3 误区3："应该用加权平均"

**错误理解**：测站数据量不同，应该按数据量加权。

**实际情况**：
- 本库中所有测站采样频率一致（年均 ~7300 条），等权 = 加权
- 即使数据量不同，只要采样频率一致，等权平均也是合理的
- 加权平均适用于**测站密度不同且有明确权重依据**的场景（如人口加权、面积加权）
