# 高程基准说明

> **适用范围**：`water-situation` skill 及所有涉及水位绝对值的场景
> **数据来源**：`sl323.st_stbprp_b.dtmel`（基面高程值）、`sl323.st_stbprp_b.dtmnm`（基准名称）
> **重要提示**：数据库中**部分测站的高程值为空**，且**未发现任何使用 1985 国家高程基准的记录**

## 2. 三种基准说明

### 2.1 废黄河口

- **定义**：以废黄河（历史上黄河夺淮入海形成的废河道）入海口的某个固定点为高程零点
- **使用区域**：江苏省里下河地区（包括扬州里运河沿线）
- **特点**：
  - 与 1985 国家高程基准存在系统差（差值约 0.19m，但**本库无可比对站点验证此差值**）
  - 本库中多个测站使用此基准（宝应、高邮等）

### 2.2 冻结(吴淞)

- **定义**：
  - **吴淞**：上海吴淞口验潮站历史平均海平面，是中国东部沿海经典高程基准之一
  - **冻结**：表示该站高程数据在某次统一校测后**锁定不再变动**，采用历史冻结值
- **使用区域**：长江流域（如大通站）
- **特点**：
  - 吴淞基准本身与 1985 国家高程基准也有系统差
  - "冻结"表示该站不随后续地面沉降/水准网更新而调整

### 2.3 1985 国家高程基准

- **定义**：中国现行法定高程基准，1987 年正式启用
- **本库实测**：**未发现任何测站使用此基准**
- **注意**：
  - ❌ 案例中"洪泽湖默认 1985 国家高程基准"的说法**与数据库不符**
  - ❌ 不要假设本库统一使用 1985 基准
  - 如需引用 1985 基准数据，需说明数据来源（如外部文件、省防指文件等）

## 3. 跨基准对比注意事项

### 3.1 基准换算

| 对比组合 | 是否可直比 | 说明 |
|---------|-----------|------|
| 废黄河口 vs 废黄河口 | ✅ **可以** | 基准相同，直接比 |
| 冻结(吴淞) vs 冻结(吴淞) | ✅ **可以** | 基准相同，直接比 |
| 废黄河口 vs 吴淞 | ❌ **不可直比** | 零点不同，需换算 |
| 废黄河口 vs 1985国家高程 | ⚠️ **待确认** | 常识差值约 0.19m，但**本库无实测数据支撑** |
| 吴淞 vs 1985国家高程 | ⚠️ **待确认** | 差值依赖外部换算表 |

### 3.2 实际场景

#### 场景 1：洪泽湖 vs 长江水位对比

```markdown
洪泽湖年均水位 13.148m（基准：废黄河口）
长江年均水位 9.528m（基准：冻结(吴淞)）

⚠️ **两站基准不同，绝对水位不可直接对比**
数值差 3.62m 包含基准系统偏差，不代表真实水位差
```

**建议处理方式**：
1. **仅报告分站水位**，标注各自的基准名称
2. **如需对比**：说明"两站基准不同，建议统一基准后再比较"
3. **不主动换算**：除非用户提供换算关系或明确要求

#### 场景 2：对外省/跨部门数据对账

```markdown
⚠️ **基准对齐提醒**
- 本库洪泽湖蒋坝站基准：废黄河口（dtmnm='废黄河口'）
- 对方数据基准：1985 国家高程基准（假设）
- 两者差值约 0.19m（常识，本库无可比对站点验证）

建议：对账前确认对方基准，或提供基准换算表。
```

## 4. 使用规则

### 4.1 查询阶段

- 默认返回数据库中原始值，不自动换算基准
- 如需查询基准信息，联表 `st_stbprp_b` 获取 `dtmnm` 字段

```sql
SELECT r.tm, r.z, b.stnm, b.rvnm, b.dtmnm
FROM sl323.st_river_r r
JOIN sl323.st_stbprp_b b ON r.stcd = b.stcd
WHERE b.stnm LIKE '%蒋坝%'
LIMIT 10
```

### 4.2 输出阶段

- **始终标注基准名称**：`水位 13.148m（基准：废黄河口）`
- **基准缺失时说明**：`水位 13.148m（基准：未标注）`
- **跨站对比时预警**：如基准不同，在报告中用 ⚠️ 标注

### 4.3 基准缺失处理

当 `dtmnm` 为空或 `dtmel` 为 NULL 时：

```markdown
水位：13.148m（基准：废黄河口，高程值未记录）
```

- **不要猜测基准**：即使知道该区域通常用某基准，也需说明"通常用XX基准，但本站未记录"
- **建议补充**：提示用户"该站基准信息缺失，建议对接省防指档案确认"

## 5. 常见误区

### 5.1 案例中的错误

**案例声称**："洪泽湖官方指标默认 1985 国家高程基准"

**数据库实测**：
- 蒋坝站 `dtmnm = '废黄河口'`（非 1985 基准）
- `dtmel = NULL`（高程值缺失）

**结论**：❌ 案例说法**与数据库不符**

### 5.2 不要假设全库统一基准

- **错误**："sl323 库统一用 1985 基准"
- **正确**："sl323 库存在多种基准（废黄河口、吴淞等），需逐站查询 dtmnm"

### 5.3 不要混淆"基准"和"水尺零点"

- **高程基准**：整个区域的统一零点（如废黄河口）
- **水尺零点**：单个水尺的读数起点，可能因水尺安装高度不同而不同
- 本库 `dtmel` 存储的是**高程基准换算后的绝对高程**，非水尺读数
