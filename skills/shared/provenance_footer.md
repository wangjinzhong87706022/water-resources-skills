# 输出溯源 Footer(Provenance Footer)

> 所有水利 skill 的查询结果**末尾必须附带**这个 footer。它让用户/下游一眼判断"这个结果该不该信、数据有多新、谁负责"——是防"静默失败(答案错但看着合理)被直接转发"的最后防线(详见 Anthropic 数据分析实践的 provenance footer)。

## 必备字段

| 字段 | 内容 | 示例 |
|------|------|------|
| **来源** | 涉及的库.表(+ 测站/范围) | `sl323.st_river_r`(古运河 3 站)JOIN `st_stbprp_b` |
| **新鲜度** | `MAX(tm)` 数据最新时间点 + 距今判断 | 数据截至 2026-06-21 08:00,距今约 1.5 天 |
| **skill 版本** | 当前 SKILL.md frontmatter 的 `version` | `water-situation v2.0.0` |
| **审查状态** | 是否经独立对抗审查 + 轮次 | ✅ 高风险,reviewer 1 轮 passed / 低风险未审查 |
| **owner**(可选) | 数据域负责人 | 水情组 / 留待各域填 |

## 新鲜度判断规则

- 查 `MAX(tm)` 作为数据最新点。
- 距今判断:
  - **< 1 天**:实时/准实时
  - **1~3 天**:有延迟,正常
  - **> 3 天** ⚠️:footer **高亮"数据陈旧"**,提示结果可能不代表当前
  - **预测表(slztk)**:最新 taskid 距今 > 1 天 → footer 标注"预测非实时"(预测系统可能多日未运行)

## 模板

**高风险查询(经独立审查)**:

```
---
📋 数据溯源
• 来源:sl323.st_river_r(古运河 3 站)JOIN st_stbprp_b
• 新鲜度:数据截至 2026-06-21 08:00(MAX(tm)),距今约 1.5 天
• skill:water-situation v2.0.0
• 审查:✅ 高风险,经独立 reviewer 1 轮 passed
---
```

**低风险查询(未审查)**:

```
---
📋 数据溯源 | sl323.st_pptn_r(stcd=58245) | 截至 2026-06-22(MAX(tm)) | rainfall v2.0.0 | 低风险·未审查
---
```

**数据陈旧** ⚠️:

```
---
📋 数据溯源 | slztk.st_mx_preset_cal_r | 最新预测 taskid 截至 2026-06-15(距今 7 天)⚠️预测非实时 | water-forecast v2.0.0 | 已审查
---
```
