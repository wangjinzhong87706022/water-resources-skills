# 水利统计分析方法

> 水利数据分析中的描述统计、趋势分析、异常检测与假设检验方法。所有水利 skill 通用。

## 一、描述统计

### 集中趋势选择

| 数据特征 | 推荐指标 | 水利场景示例 |
|---------|---------|-------------|
| 对称分布、无异常值 | 平均值 (mean) | 同一测站多年同期水位 |
| 偏态分布 | 中位数 (median) | 日降雨量分布（多数无雨日拉偏） |
| 分类或有序数据 | 众数 (mode) | 水质等级分布、水位涨落趋势码 |
| 有极端值（如洪峰流量） | 中位数 + 平均值 同时报告 | 两者差距越大，偏态越严重 |

### 百分位体系

```
p1:   下限 / 最低典型值
p5:   正常范围低端
p25:  第一四分位
p50:  中位数（典型值）
p75:  第三四分位
p90:  高值范围（如高水位场景）
p95:  正常范围高端
p99:  极端值（如历史最高水位）
```

**水利示例**："该站中位数水位 3.2m，但 p90 达 6.8m，p99 达 9.5m（超保证水位），均值被拉高到 4.1m——说明日常水位平稳，极端洪水时水位急剧抬升。"

### 分布特征描述

对每个数值型数据列，描述：
- **形状**：正态 / 右偏 / 左偏 / 双峰 / 均匀 / 重尾
- **中心**：均值和中位数（及其差距）
- **离散度**：标准差或 IQR
- **异常值**：数量和极端程度
- **边界**：是否有自然下限（水位=0m）或上限（降雨不可能为负）

**Python 快速画像**：
```python
import pandas as pd
import numpy as np

def profile_series(s: pd.Series, name: str = '指标'):
    desc = s.describe(percentiles=[.01, .05, .25, .5, .75, .9, .95, .99])
    skew = s.skew()
    print(f"=== {name} 统计画像 ===")
    print(f"样本数: {desc['count']:.0f}")
    print(f"均值±标准差: {desc['mean']:.3f} ± {desc['std']:.3f}")
    print(f"中位数 [p25-p75]: {desc['50%']:.3f} [{desc['25%']:.3f} - {desc['75%']:.3f}]")
    print(f"p1-p99 范围: {desc['1%']:.3f} - {desc['99%']:.3f}")
    print(f"偏度: {skew:.3f} (正=右偏, 负=左偏)")
    return desc
```

## 二、趋势分析

### 移动平均

```python
# 7 日移动平均（适用于逐日水位/流量数据）
df['ma_7d'] = df['z'].rolling(window=7, min_periods=1).mean()

# 30 日移动平均（月尺度平滑）
df['ma_30d'] = df['z'].rolling(window=30, min_periods=1).mean()
```

### 同比/环比比较

| 对比类型 | 水利应用场景 | 说明 |
|---------|-------------|------|
| 周同比 (WoW) | 对比本周某日与上周同日水位 | 受气象波动影响大 |
| 月环比 (MoM) | 逐月降雨量/来水量比较 | 适合趋势跟踪 |
| 年同比 (YoY) | 今年同期 vs 去年同期的水位/降雨/水质 | 消除季节性，黄金标准 |
| 多年均值对比 | 当前水位 vs 历史同期均值 | 判断丰枯偏离程度 |

### 增长率计算

```
简单增长率: (当前值 - 基期值) / 基期值
水位变率: (z₂ - z₁) / (t₂ - t₁)  单位: m/h 或 m/d
水文距平: (当前值 - 历史同期均值) / 历史同期标准差
```

### 季节性检测

对水位/降雨/水质时间序列，检查：
1. **日尺度**：是否存在日内规律（水质 DO 有日变化，水位一般无）
2. **周尺度**：有没有周末效应（水质监测频率差异）
3. **月尺度**：汛期 vs 非汛期规律（6-9月 vs 其他月）
4. **年尺度**：丰水年 vs 枯水年周期

**始终用年同比消除季节性**——"今年6月降雨 vs 去年6月降雨"，而非"今年6月 vs 今年5月"。

### 水位变化率（涨落速率）

> `LAG() OVER` 是 MySQL **8.0+** 特性，5.7 用自连接找"上一条"。

```sql
-- ① 当前基线（5.7）：自连接
SELECT cur.stcd, cur.tm, cur.z,
       ROUND((cur.z - prev.z) /
             NULLIF(TIMESTAMPDIFF(HOUR, prev.tm, cur.tm), 0), 3) AS z_change_rate_mh
FROM st_river_r cur
LEFT JOIN st_river_r prev
  ON prev.stcd = cur.stcd
 AND prev.tm = (SELECT MAX(tm) FROM st_river_r p
                WHERE p.stcd = cur.stcd AND p.tm < cur.tm)
WHERE cur.stcd = '50801450' AND cur.tm >= DATE_SUB(NOW(), INTERVAL 3 DAY)
ORDER BY cur.tm;
```
```sql
-- ② 现代写法（8.0+）
SELECT stcd, tm, z,
       (z - LAG(z,1) OVER (PARTITION BY stcd ORDER BY tm)) /
       NULLIF(TIMESTAMPDIFF(HOUR, LAG(tm,1) OVER (PARTITION BY stcd ORDER BY tm), tm), 0)
       AS z_change_rate_mh
FROM st_river_r
WHERE stcd = '50801450' AND tm >= DATE_SUB(NOW(), INTERVAL 3 DAY)
ORDER BY tm;
```

## 三、异常检测

### 统计方法

**Z-score 法**（适用于正态分布数据）：
```python
z_scores = (df['z'] - df['z'].mean()) / df['z'].std()
outliers = df[abs(z_scores) > 3]  # 超过3个标准差
```

**IQR 法**（对非正态分布更鲁棒）：
```python
Q1 = df['z'].quantile(0.25)
Q3 = df['z'].quantile(0.75)
IQR = Q3 - Q1
lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR
outliers = df[(df['z'] < lower) | (df['z'] > upper)]
```

**百分位法**（最简单）：
```python
outliers = df[(df['z'] < df['z'].quantile(0.01)) |
              (df['z'] > df['z'].quantile(0.99))]
```

### 水利数据异常识别

| 数据类型 | 异常条件 | 可能原因 | 处理方法 |
|---------|---------|---------|---------|
| 水位 (z) | < 0m（内陆站） | 传感器故障、干涸 | 标注，区分干涸 vs 故障 |
| 水位变率 (Δz/Δt) | > 2m/h | 传感器漂移、闸门突变 | 与相邻站交叉验证 |
| 降雨量 (drp) | 单日 > 500mm 或 < 0mm | 传感器故障 | 与邻近站比较 |
| 溶解氧 (DO) | > 20mg/L 或 < 0mg/L | 传感器校准问题 | 标记为可疑 |
| CODMn | > 50mg/L | 严重污染或录入错误 | 与历史对比确认 |
| 流量 (q) | 负值 | 传感器反向、数据异常 | 标记不可用 |
| 时间戳 (tm) | 未来时间或早于 1970 年 | 时钟不同步 | 排除或修正 |

### 处理原则

**不自动删除异常值**。按以下流程处理：
1. **调查**：数据错误、真实极端值、还是不同数据总体？
2. **数据错误**：修复或排除（如日期错乱、传感器突跳）
3. **真实极端值**：保留，改用鲁棒统计（中位数代替均值）
4. **不同总体**：分拆单独分析（如非汛期 vs 汛期）

报告处理方式："共标记水位突跳 12 条记录（0.3%），经与相邻 3 站交叉验证确认为传感器干扰，已排除。"

## 四、假设检验基础

### 适用场景

| 场景 | 方法 | 水利案例 |
|------|------|---------|
| 两组均值比较 | t 检验 | 同比是否比去年显著偏高 |
| 两组比例比较 | z 检验 | 水质超标率是否比去年改善 |
| 同站前后比较 | 配对 t 检验 | 治理前后水质是否显著变化 |
| 三组以上比较 | ANOVA | 不同河段水质是否有差异 |
| 非正态比较 | Mann-Whitney U | 降雨量偏态分布下比较两站差异 |
| 分类关联 | 卡方检验 | 季节与水质等级是否有相关性 |

### 实践意义 ≠ 统计显著

大样本下微小差异也会显著。始终报告：
- **效应量**：差距有多大？（"V类断面占比下降 3.2 个百分点"）
- **业务意义**："从 5.2% 降到 2.0%，相当于减少 12 个超标断面"

### 水利场景特别注意事项

| 陷阱 | 说明 |
|------|------|
| **辛普森悖论** | 每一分段水质都改善，但总体恶化——因为采样向重污染河段倾斜 |
| **幸存者偏差** | "还在运行的站点水位正常"——故障站已被排除（不报送数据） |
| **样本不独立** | 同一河段上下游数据有空间自相关，不能视作独立样本 |
| **多重比较** | 同时比较 20 个站时，约 1 个会"假显著" |
| **伪相关** | 两变量同步上升 ≠ 因果关系（需水文机理印证） |
| **虚假精度** | "水位 3.1416m" 暗示精确到 0.1mm，实际传感器精度仅 1cm → ROUND(z, 2) |

## 五、水利统计应用速查

| 分析需求 | 方法 | 关键参数 |
|---------|------|---------|
| 描述当前水位分布 | 百分位 + 均值标准差 | p25/p50/p75 |
| 判断水位是否超往年 | Z-score（相对历史均值） | |z| > 2 |
| 检测降雨突变 | 滑动 t 检验 / Pettitt 检验 | 年序列 |
| 对比两站水质差异 | t 检验或 Mann-Whitney U | 看数据是否正态 |
| 判断水质改善是否显著 | 配对 t 检验（同站前后） | 治理前后匹配 |
| 预测短期水位趋势 | 移动平均 + 线性外推 | 周期选择 |
| 识别异常水位过程 | IQR 法 | 1.5×IQR |
| 评估预报精度 | NSE（纳什效率系数）、RMSE | NSE > 0.7 可接受 |
